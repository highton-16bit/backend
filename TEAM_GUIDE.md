# 📸 Plog (Photo + Log) 통합 명세서

본 문서는 **Plog** 서비스의 핵심 도메인 모델과 주요 기능의 동작 원리를 정의한다. 모든 API 요청 시 `Authorization: {username}` 헤더를 통해 유저를 식별한다.

---

## 1. 핵심 도메인 엔티티 (Domain Entities)

### 1.1 여행 (Travel) & 일정 (PlanItem)
*   **여행 (Travel):** 서비스의 최상위 데이터 단위로, 일정과 사진의 부모 엔티티이다.
*   **일정 (PlanItem):** 여행 기간 내에 발생하는 상세 계획 데이터이다.
    *   **필드:** 날짜(Date), 시작시간(StartTime), 종료시간(EndTime), 메모(Memo).

### 1.2 스냅샷 (Snapshot)
*   **정의:** 여행 중 특정 시점에 기록된 실시간 이미지 리소스이다.
*   **필드:** 이미지 URL(ImageUrl), 스냅샷 여부(isSnapshot: true).

### 1.3 게시글 (Post) & 소셜 기능
*   **정의:** 완료된 여행을 기반으로 생성되는 소셜 공유 단위이다.
*   **신규 필드:**
    *   `likeCount`: 사용자들이 누른 좋아요 총합.
    *   `cloneCount`: 해당 게시글의 일정을 자신의 여행으로 복사해간 횟수. (인기 지표)
*   **북마크 (Bookmark):** 사용자가 나중에 다시 보기 위해 저장한 게시물 목록.

---

## 2. 주요 기능 명세 (Functional Specifications)

### 2.1 여행 일정 텍스트 변환 (Flattening)
*   게시글 생성 시, 백엔드는 해당 여행의 모든 일정을 정적 텍스트 리스트로 변환하여 `ContentSummary`에 저장한다.

### 2.2 스마트 클로닝 (AI Smart Clone)
*   타인의 게시글 요약을 AI가 분석하여 JSON 데이터로 추출한다. **추출 성공 시 해당 게시글의 `cloneCount`가 1 증가한다.**

### 2.3 좋아요 & 북마크 토글 (Social Toggle)
*   **좋아요:** 한 번 누르면 등록, 다시 누르면 취소되는 토글 방식으로 작동한다.
*   **북마크:** 마음에 드는 게시글을 내 북마크 리스트에 보관하거나 제거할 수 있다.

---

## 3. 서비스 데이터 흐름 (Data Flow)
1.  **인증:** `POST /auth` 유저 등록 후 헤더에 `Authorization` 포함.
2.  **기록:** 여행 생성 및 상세 일정, 사진 등록.
3.  **공유:** 여행 종료 후 게시글 생성 및 일정 텍스트 변환.
4.  **소셜:** 타인의 글에 좋아요/북마크를 남기고, `Smart Clone`으로 일정을 복사(Clone)한다.
