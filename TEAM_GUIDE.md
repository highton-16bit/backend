# 📸 Plog (Photo + Log) 통합 명세서

본 문서는 Plog 서비스의 도메인 지식, 데이터 구조, 그리고 탭바별 상세 기능을 정의한다. 모든 API 요청 시 `Authorization: {username}` 헤더를 통해 유저를 식별한다.

---

## 1. 핵심 도메인 모델 (Domain Entities)

### 1.1 여행 (Travel)
*   **정의:** 서비스의 최상위 데이터 단위이며, **일정표**인 동시에 **사진첩**의 역할을 겸한다.
*   **구성:** 제목, 시작일, 종료일, 지역명.
*   **특징:** 하위에 N개의 상세 일정(PlanItem)과 N개의 사진(Photo)을 소유한다.

### 1.2 일정 (PlanItem)
*   **정의:** 여행 내의 시간대별 계획이다.
*   **구성:** 날짜, 시작시간, 종료시간, 메모(Memo).
*   **특징:** 별도의 장소 필드 없이 메모에 모든 내용을 기록하며, 게시글 생성 시 텍스트 변환의 원천 데이터가 된다.

### 1.3 사진 (Photo)
*   **정의:** 여행 중 수집된 모든 시각적 기록이다.
*   **구분:**
    *   **일반 사진 (Gallery):** 사용자가 여행 사진첩에 직접 업로드한 사진.
    *   **스냅샷 (Snapshot):** 여행 중 무작위 알림을 통해 기록된 실시간 사진 (`isSnapshot: true`).
*   **특징:** 여행 엔티티에 귀속되어 해당 여행의 '디지털 앨범'을 구성한다.

### 1.4 게시글 (Post)
*   **정의:** 완료된 여행의 기록을 외부로 공유하는 단위이다.
*   **구성:** 제목, 요약 텍스트(ContentSummary), 선택된 사진 리스트, 소셜 지표(좋아요, 클론 수).

## 2. 탭바별 상세 기능 명세 (Feature by Tabs)

### Tab 1. 홈 (Home): 현재의 기록
*   **클라이언트:**
    *   현재 날짜 기준 `진행 중`인 여행 정보를 최상단에 노출한다.
    *   진행 중인 여행이 없을 경우, `가장 가까운 예정된 여행` 혹은 `최근 종료된 여행`을 추천한다.
    *   스냅샷 촬영 알림(로컬 푸시)을 처리하고 촬영 화면을 제공한다.
*   **백엔드:**
    *   현재 날짜와 여행 기간을 비교하여 우선순위에 따른 단일 여행 정보를 반환한다.

### Tab 2. 탐색 (Discovery): 행복의 발견
*   **클라이언트:**
    *   전체 사용자의 게시글 피드를 무한 스크롤 형태로 노출한다.
    *   AI 검색창을 통해 자연어 질문을 던지고 답변을 확인한다.
    *   게시글 상세 보기에서 '스마트 클로닝' 버튼을 통해 일정을 복사한다.
*   **백엔드:**
    *   **AI 검색:** DB 게시글 키워드 검색과 Gemini 지식을 결합하여 답변을 생성한다.
    *   **소셜 로직:** 좋아요/북마크 토글 및 게시글별 클론 횟수를 관리한다.

### Tab 3. 새로 만들기 (New): 계획과 공유
*   **클라이언트:**
    *   **여행 생성:** 신규 여행 정보(제목, 기간 등)를 입력한다.
    *   **게시글 생성:** 완료된 여행 중 하나를 선택하고, 해당 여행의 사진첩(스냅샷 포함)에서 공유할 사진들을 선택한다.
*   **백엔드:**
    *   **Flattening:** 게시글 생성 요청 시, 선택된 여행의 모든 일정을 읽어 "시간: 메모" 형태의 정적 텍스트로 변환하여 저장한다.

### Tab 4. 여행 (Travels): 나의 사진첩
*   **클라이언트:**
    *   내가 생성한 모든 여행 목록을 리스트/그리드 형태로 보여준다.
    *   특정 여행을 클릭하면 **사진첩 뷰**와 **캘린더 뷰**를 전환하며 볼 수 있다.
    *   여행 도중 언제든지 새로운 사진을 업로드하여 사진첩을 채울 수 있다.
*   **백엔드:**
    *   여행별 사진 목록 및 상세 일정 목록을 반환한다.
    *   사진 업로드 프록시(수파베이스 연동)를 제공한다.

### Tab 5. 프로필 (Profile): 나의 기록
*   **클라이언트:**
    *   내 활동 통계(총 여행 횟수, 수집한 스냅샷 수, 받은 좋아요 등)를 시각화한다.
    *   내가 북마크한 게시글 목록을 확인할 수 있는 진입점을 제공한다.
*   **백엔드:**
    *   유저별 활동 데이터 집계 및 북마크 리스트를 반환한다.

## 3. 핵심 비즈니스 로직 (Logic Deep Dive)

### 3.1 사진 선택 및 게시 (Photo Selection)
*   게시글 작성 시 사용자는 해당 여행에 등록된 모든 사진(일반 사진 + 스냅샷) 중 원하는 것을 다중 선택할 수 있다.
*   선택된 사진들은 `PostPhotoMappings` 테이블을 통해 게시글과 연결된다.

### 3.2 스마트 클로닝 (Smart Clone)
*   타인의 게시글 본문(텍스트)을 AI가 분석하여 `date, startTime, endTime, memo` 형태의 구조화된 데이터로 변환한다.
*   사용자가 수락하면, 백엔드는 해당 데이터를 기반으로 **신규 여행(Travel)과 일정(PlanItems)을 자동 생성**한다.

### 3.3 인증 및 식별 (Auth)
*   모든 API는 `Authorization` 헤더에 담긴 `username` 문자열을 고유 식별자로 사용한다.
*   최초 1회 `POST /auth` 호출을 통해 유저 엔티티를 생성한다.
