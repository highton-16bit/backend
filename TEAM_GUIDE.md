# 📸 Plog (Photo + Log) 통합 명세서

본 문서는 Plog 서비스의 도메인 지식, 데이터 구조, 그리고 탭바별 상세 기능을 정의한다. 모든 API 요청 시 `Authorization: {username}` 헤더를 통해 유저를 식별한다.

---

## 1. 핵심 도메인 모델 (Domain Entities)

### 1.1 여행 (Travel)
*   **정의:** 서비스의 최상위 데이터 단위이며, 일정표인 동시에 사진첩의 역할을 겸한다.
*   **구성:** 제목, 시작일, 종료일, 지역명.
*   **특징:** 하위에 N개의 상세 일정(PlanItem)과 N개의 사진(Photo)을 소유한다.

### 1.2 일정 (PlanItem)
*   **정의:** 여행 내의 시간대별 계획이다.
*   **구성:** 날짜, 시작시간, 종료시간, 메모(Memo).
*   **특징:** 별도의 장소 필드 없이 메모에 모든 내용을 기록하며, 게시글 생성 시 텍스트 변환의 원천 데이터가 된다.

### 1.3 사진 (Photo)
*   **정의:** 여행 중 수집된 모든 시각적 기록이다.
*   **구분:**
    *   **일반 사진 (Gallery):** 사용자가 여행 사진첩에 직접 업로드한 사진.
    *   **스냅샷 (Snapshot):** 여행 중 무작위 알림을 통해 기록된 실시간 사진 (`isSnapshot: true`).
*   **특징:** 여행 엔티티에 귀속되어 해당 여행의 '디지털 앨범'을 구성한다.

### 1.4 게시글 (Post)
*   **정의:** 완료된 여행의 기록을 외부로 공유하는 단위이다.
*   **구성:** 제목, 요약 텍스트(ContentSummary), 선택된 사진 리스트, 소셜 지표(좋아요, 클론 수).

## 2. 탭바별 상세 기능 명세 (Feature by Tabs)

### Tab 1. 홈 (Home)
*   **클라이언트 UI/UX:**
    *   **상단:** 현재 날짜 기준 `진행 중`인 여행 정보를 노출한다. (진행 중인 여행이 없을 경우 가장 가까운 예정/종료 여행 추천)
    *   **하단:** 인스타그램 홈 피드 스타일의 **게시글 상세 뷰 무한 스크롤**을 배치한다. (전체 공개된 게시글들을 카드 형태로 노출)
    *   스냅샷 촬영 알림 처리 및 촬영 화면 진입.
*   **백엔드:**
    *   현재 날짜 기반 단일 여행 정보 반환.
    *   전체 게시글 목록을 상세 데이터(사진, 내용 포함)와 함께 페이징하여 반환.

### Tab 2. 탐색 (Discovery)
*   **클라이언트 UI/UX:**
    *   인스타그램 탐색 탭 스타일의 **그리드 뷰 무한 스크롤**을 노출한다. (각 게시글의 첫 번째 이미지만 노출)
    *   특정 이미지 클릭 시 해당 **게시글 상세 보기 뷰**로 이동한다.
    *   AI 검색창을 통해 자연어 질문 및 답변 확인.
*   **백엔드:**
    *   전체 게시글의 썸네일(첫 번째 사진) 목록을 페이징하여 반환.
    *   AI 검색: DB 키워드 검색과 Gemini 지식을 결합한 답변 생성.

### Tab 3. 새로 만들기 (New)
*   **클라이언트 UI/UX:**
    *   **여행 생성:** 신규 여행 정보(제목, 기간 등) 입력.
    *   **게시글 생성:** 완료된 여행 선택 -> 해당 여행의 사진첩(스냅샷 포함)에서 공유할 사진 다중 선택 -> 게시.
*   **백엔드:**
    *   **Flattening:** 게시글 생성 시, 선택된 여행의 일정을 "시간: 메모" 형태의 텍스트로 자동 변환 저장.

### Tab 4. 여행 (Travels)
*   **클라이언트 UI/UX:**
    *   내가 생성한 여행 목록을 리스트/그리드 형태로 제공.
    *   특정 여행 클릭 시 **사진첩(Gallery) 뷰**와 **캘린더(Plan) 뷰** 제공.
    *   여행 도중 상시 사진 업로드 기능.
*   **백엔드:**
    *   여행별 사진/일정 목록 반환 및 사진 업로드 프록시 처리.

### Tab 5. 프로필 (Profile)
*   **클라이언트 UI/UX:**
    *   내 활동 통계(여행 횟수, 스냅샷 수 등) 시각화.
    *   내가 북마크한 게시글 목록 확인.
*   **백엔드:**
    *   유저 활동 데이터 집계 및 북마크 리스트 반환.

## 3. 핵심 비즈니스 로직 (Logic Deep Dive)

### 3.1 게시글 상세 뷰 (Post Detail View)
*   홈 탭 하단과 탐색 탭 클릭 시 연결되는 공통 뷰.
*   게시글 제목, 사진 리스트(슬라이더), AI로 변환된 일정 텍스트, 좋아요/북마크 버튼, 클론 버튼 포함.

### 3.2 스마트 클로닝 (Smart Clone)
*   타인의 게시글 본문(텍스트)을 AI가 분석하여 `date, startTime, endTime, memo` 형태의 구조화된 데이터로 변환.
*   클로닝 시 백엔드에서 신규 여행(Travel) 및 일정(PlanItems) 자동 생성.

### 3.3 인증 및 식별 (Auth)
*   `Authorization: {username}` 헤더 기반 고유 식별.
*   최초 1회 `POST /auth`를 통한 자동 회원가입.
