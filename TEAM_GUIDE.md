# 📸 Plog (Photo + Log) 통합 명세서

본 문서는 **Plog** 서비스의 핵심 도메인 모델과 주요 기능의 동작 원리를 정의한다. 모든 API 요청 시 `Authorization: {username}` 헤더를 통해 유저를 식별한다.

---

## 1. 핵심 도메인 엔티티 (Domain Entities)

### 1.1 여행 (Travel)
*   **정의:** 서비스의 최상위 데이터 단위이며, 일정과 사진의 부모 엔티티이다.
*   **구성 요소:** 제목(Title), 시작일(StartDate), 종료일(EndDate), 지역명(Region).
*   **특징:** 시스템 현재 시간과 여행 기간을 비교하여 `진행 중`, `예정`, `종료` 상태를 프론트엔드에서 연산하여 활용한다.

### 1.2 일정 (PlanItem)
*   **정의:** 여행 기간 내에 발생하는 상세 계획 데이터이다.
*   **구성 요소:** 날짜(Date), 시작시간(StartTime), 종료시간(EndTime), 메모(Memo), 순서(OrderIndex).
*   **특징:** 별도의 장소명 필드 없이 **메모(Memo)** 필드에 장소와 활동 내역을 통합하여 기록한다.

### 1.3 스냅샷 (Snapshot)
*   **정의:** 여행 중 특정 시점에 기록된 실시간 이미지 리소스이다.
*   **구성 요소:** 이미지 URL(ImageUrl), 스냅샷 여부(isSnapshot).
*   **동작:** 일반 업로드 사진과 달리 `isSnapshot: true` 플래그를 통해 실시간 기록 데이터임을 명시한다.

### 1.4 게시글 (Post)
*   **정의:** 완료된 여행을 기반으로 생성되는 소셜 공유 단위이다.
*   **구성 요소:** 제목(Title), 요약 내용(ContentSummary), 좋아요 수(LikeCount), 사진 매핑.

---

## 2. 주요 기능 명세 (Functional Specifications)

### 2.1 여행 일정 텍스트 변환 (Flattening)
*   **목적:** 게시글 생성 시 복잡한 일정 데이터를 하나의 읽기 쉬운 텍스트로 고정한다.
*   **방식:** 사용자가 게시글 생성을 요청하면, 백엔드는 해당 여행의 모든 `PlanItem`을 날짜/시간 순으로 정렬하여 정적 텍스트 리스트를 생성한다.
*   **데이터 저장:** 생성된 텍스트는 `Post` 엔티티의 `ContentSummary` 필드에 저장된다. (AI를 사용하지 않는 정적 변환 로직)
    *   *포맷 예시:* "2026-02-02 일정: - 16:00 ~ 17:00 : 비행기 탑승"

### 2.2 스마트 클로닝 (AI Smart Clone)
*   **목적:** 타인의 게시글에 적힌 텍스트 정보를 나의 새로운 여행 일정 데이터로 복원한다.
*   **동작 흐름:**
    1.  사용자가 특정 게시글의 '클로닝' 버튼을 누른다.
    2.  백엔드는 해당 게시글의 `ContentSummary`(텍스트)를 **AI(Gemini)**에게 전달한다.
    3.  AI는 텍스트를 분석하여 `date, startTime, endTime, memo` 필드를 가진 **JSON 배열**로 추출하여 반환한다.
*   **활용:** 프론트엔드는 반환된 JSON 데이터를 기반으로 사용자의 새 여행 일정을 즉시 생성하거나 수정 인터페이스를 제공한다.

### 2.3 AI 하이브리드 검색 (Search)
*   **목적:** 자연어 쿼리를 통해 최적의 여행 장소와 관련 게시글을 탐색한다.
*   **방식:** 사용자의 질문을 입력받아 내부 DB의 `Post` 데이터 내 키워드 검색 결과와 AI의 내장 지식을 결합하여 요약된 답변과 관련 게시글 리스트를 함께 반환한다.

---

## 3. 서비스 데이터 흐름 (Data Flow)

1.  **인증:** `POST /auth`를 통해 유저 등록 후, 모든 요청 헤더에 `Authorization: {username}`을 포함한다.
2.  **계획 & 기록:** 여행(`Travel`)을 생성하고, 상세 일정(`PlanItem`)과 사진(`Snapshot`)을 등록한다.
3.  **공유:** 여행을 종료하고 게시글(`Post`)을 생성하면, 시스템이 일정을 텍스트로 변환하여 저장한다.
4.  **재활용:** 탐색 탭에서 마음에 드는 게시글을 발견하면 `Smart Clone`을 통해 해당 일정을 내 여행으로 가져온다.
